Final Project Assignment: AI Portfolio Website \(Django\)

# Overview

For your final project, you will create a professional AI Portfolio Website using Django with Tailwind\.
This website will serve as a showcase of your work throughout the course and will be used during your mock interview with industry professionals\.

The goal is to demonstrate not only what you built, but your ability to explain your work, reflect on challenges, and communicate business value\.

# Learning Objectives

By completing this project, you will:
\- Build a full Django web application
\- Organize and present technical projects professionally
\- Demonstrate understanding of AI tools and workflows
\- Communicate technical concepts clearly
\- Prepare for real\-world technical interviews

# Project Requirements

Your website must include the following pages:
\- Home Page
\- About Page
\- Projects Page
\- Individual Project Detail Pages
\- Skills Page
\- Blog Page
\- Resume Page
\- Contact Page

Your site must use Django models, views, templates, and authentication where appropriate\.

# Required Projects to Include

You must include the following projects:
\- Chatbot Project
\- n8n Agent Workflow Project
\- LangChain Agent Project
\- Google AI Studio Media Project
\- Machine Learning Project \(scikit\-learn\)
\- Campus SkillSwap Django Project

Each project must have its own detail page\.

# Project Detail Page Structure

Each project page must include:
1\. Project Title
2\. One\-Sentence Summary
3\. Business Problem
4\. Tools Used
5\. Key Features
6\. Your Role / Contribution
7\. Biggest Challenge
8\. What You Learned
9\. Screenshot or Visual
10\. GitHub or Demo Link \(if available\)

# Database Requirements

You must create at least one Django model for storing project information\.

Suggested fields:
\- title
\- summary
\- description
\- category
\- tools\_used
\- challenges
\- lessons\_learned
\- image
\- links

Additional models \(optional\):
\- Skills
\- Experience

# Design Expectations

Your website should:
\- Be clean and professional
\- Use consistent styling \(Bootstrap recommended\)
\- Be easy to navigate
\- Clearly present information
\- Include visual elements where appropriate

# Mock Interview Component

During your mock interview, you will:
\- Walk through your portfolio website
\- Explain your projects in detail
\- Discuss challenges and solutions
\- Answer questions from industry professionals

# Submission Instructions

Submit:
\- GitHub repository link
\- Live deployment link \(AWS or Render\)

Be prepared to present your website during your assigned mock interview time\.
